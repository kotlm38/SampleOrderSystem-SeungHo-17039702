## 시스템 구성 요소
1. 주문
- 모든 주문은 5가지 Status(Reserved, Rejected, Producing, Confirmed, Release)중 하나를 보유한다.
- Reserved : 주문이 입력된 초기 상태.
- Rejected : 주문이 거절된 상태. 더이상 관리하지 않고 주문 목록에서 삭제.
- Confirmed : 주문에 해당하는 시료 준비가 완료된 상태.
- Release : 시료를 고객에게 전달하고 주문 처리가 완료된 상태.
2. 시료
- 시료별로 5가지 속성(ID, 이름, 평균생산시간, 수율, 재고)을 가진다.
- 각 시료는 고유의 ID를 가지고 ID에 해당하는 이름은 하나이다.
- 시료의 생산량은 부족분/(수율*0.9) 으로 정해진다.
- 총 생산 시간은 평균생산시간*생산량 으로 정해진다.
- 생산은 한 종류의 시료씩 이루어지며 주문 입력 순서대로(FIFO방식으로) 처리한다. 

## 주문의 처리 순서
1. 주문이 접수되면 reserved 상태로 등록되며 승인여부 결정 대기.
2. 거절이 된다면 즉시 주문 data삭제. 승인이 된다면 재고 확인하여 다음 동작 결정.
2-1. 재고가 충분하다면 confirmed 상태로 변경
2-2. 재고가 부족하다면 Producing 상태로 변경하고 부족분만큼 생산을 시작. 생산이 완료되면 confirmed 상태로 변경.
3. 출고실행이 되면 release상태로 변경

## Main menu
1. 시료관리 : 새로운 시료 등록, 목록조회, 이름검색기능
2. 주문(접수/ 승인/ 거절) : 고객 주문접수 및 생산담당자의 승인·거절처리
3. 모니터링 : 상태별 주문수 및 시료별 재고현황 확인
4. 출고처리 : CONFIRMED 상태 주문에 대해 출고 실행
5. 생산라인 : 현재 생산중인 시료 및 대기중인 생산큐 확인

### 시료관리
- 3개의 sub menu를 출력. 시료등록/시료조회/시료검색.
1. 시료등록 : 새로운 시료를 시스템에 추가. 속성값으로 (시료ID, 이름, 평균 생산시간, 수율) 입력.
2. 시료조회 : 등록된 모든 시료 목록을 확인. 현재 재고 수량도 같이 표시.
3. 시료검색 : 이름 등 속성으로 특정 시료를 검색.

### 주문(접수/ 승인/ 거절)
- Reserved 상태의 주문 출력. 승인/거절을 선택할 수 있도록 제공.
- 승인/거절 동작이 되면 display update.

### 모니터링
- 2개의 sub menu 출력. 주문량확인/재고량확인.
1. 주문량확인 : 상태별(Reserved, Producing, Confirmed, Release) 목록을 확인.
2. 재고량확인 : 각 시료별 현재 재고 수량 확인. 주문대비(Reserved, Producing 합산) 수량에 따라 상태도 3가지로(여유, 부족, 고갈) 표시. 고갈은 수량이 0인경우.

### 출고처리
- confirmed 상태의 주문만 display. 사용자가 주문을 선택하여 출고 실행.

### 생산라인
- 현재 생산중인 주문을 출력. 주문번호, 시료이름, 주문량, 재고, 필요 생산량, 완료 예정 시간.
- 대기중인 주문 실행 순서대로 3개 출력. 순서, 주문번호, 시료이름, 주문량, 필요 생산량, 완료 예정 시간.

## data structure
- 시료 : ID, 이름, 평균생산시간, 수율, 재고.
- 주문 : 번호, 고객, 시료, 수량, 상태(Reserved, Rejected, Producing, Confirmed, Release).
